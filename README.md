# 家谱系统

一个基于 Tkinter 的家谱可视化与编辑工具。

程序以 Canvas 为主视图，展示人物节点和关系线，支持新增、编辑、删除与关系维护，数据持久化到本地 JSON 文件。

## 功能概览

1. 家谱可视化：父母在上、本人与同辈在中、子女在下。
2. 关系线展示：
	- 父母/子女：红色带箭头实线
	- 配偶：蓝色虚线
	- 兄弟姐妹：黄色实线
3. 节点交互：
	- 双击节点设为中心人物
	- 悬停显示人物详情气泡
	- 左键拖拽移动节点（局部调整布局）
4. 右键菜单：
	- 右键节点：设为中心、编辑、删除、添加父亲、添加母亲、添加子女
	- 右键关系线：删除该关系（带确认弹窗）
5. 搜索功能：按姓名、性别、出生年代、配偶、子女、兄妹进行筛选。
6. 数据管理：
	- 启动时优先加载 data.json
	- 首次无数据自动生成示例家谱
	- 每次修改自动保存

## 项目结构

1. main.py：程序启动入口（创建 Tk 窗口并启动应用）。
2. app.py：Tkinter 界面与交互逻辑（GenealogyApp）。
3. model.py：数据模型与全局常量（Person、画布尺寸等）。
4. storage.py：数据持久化与样例数据生成（load_data/save_data）。
5. data.json：家谱数据文件（运行后自动生成或更新）。
6. assets/gender_icons.png：性别图标资源（存在时用于节点头像）。
7. README.md：项目文档。

## 数据模型

核心实体为 Person，主要字段：

1. id：人物唯一标识（UUID）
2. name：姓名
3. gender：性别（男/女）
4. birth：出生日期
5. death：去世日期
6. father：父亲人物 id
7. mother：母亲人物 id
8. spouses：配偶人物 id 列表
9. bio：生平简介

说明：

1. 子女和兄弟姐妹不单独存储，由 father/mother 动态推导。
2. 配偶关系保持双向一致（A 配偶含 B，B 配偶也含 A）。

## 使用说明

1. 视图操作：
	- 鼠标滚轮：缩放
	- 左键拖拽节点：移动节点
	- 点击“重置中心”：回到根人物视角
2. 节点操作：
	- 右键节点弹出菜单进行编辑与关系扩展
3. 关系操作：
	- 右键关系线，点击“删除该关系”
	- 二次确认后执行删除
4. 搜索人物：
	- 工具栏点击“搜索人物”打开筛选窗口
	- 双击搜索结果可设为中心人物

## 数据文件说明

1. 程序会自动将数据保存到 data.json。
2. 如需恢复内置示例数据，可先备份并删除 data.json 后重启程序。
3. 项目中的 data.json.bak-* 文件为历史备份，可用于手工恢复。

## 当前特性边界

1. 兄弟姐妹关系由共享父母推导，删除“兄弟姐妹线”本质上是拆除共享父/母关系。
2. 画布是当前中心人物的局部关系视图，不是全族谱一次性平铺。
